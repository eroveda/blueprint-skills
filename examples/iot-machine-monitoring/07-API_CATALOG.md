# API Catalog — Industrial Machine Monitoring System

This system exposes two interface types:

1. **REST API + WebSocket** — consumed by the web dashboard and third-party integrations
2. **MQTT Message Interface** — consumed by ESP32 sensor devices

---

## Base URL

| Environment | REST API | WebSocket | MQTT Broker |
|---|---|---|---|
| Production | `https://api.example.com` | `wss://api.example.com/ws` | `mqtts://mqtt.example.com:8883` |
| Staging | `https://api-staging.example.com` | `wss://api-staging.example.com/ws` | `mqtts://mqtt-staging.example.com:8883` |
| Local | `http://localhost:8000` | `ws://localhost:8000/ws` | `mqtt://localhost:1883` |

## Authentication

### How to Authenticate (REST API)

1. Obtain an access token by calling `POST /api/auth/login` with email and password
2. Include the token in every request: `Authorization: Bearer {access_token}`
3. When the access token expires (after 1 hour), refresh it using `POST /api/auth/refresh`
4. If the refresh token is also expired (after 7 days), log in again

### JWT Token Claims

| Claim | Type | Example | Description |
|---|---|---|---|
| `sub` | UUID | `"d290f1ee-6c54-4b01-90e6-d701748f0851"` | User ID |
| `email` | string | `"engineer@factory.com"` | User email |
| `role` | string | `"engineer"` | One of: `admin`, `engineer`, `viewer` |
| `iat` | integer | `1706000000` | Token issued at (Unix timestamp) |
| `exp` | integer | `1706003600` | Token expires at (Unix timestamp) |

### Authentication Flows


### How to Authenticate (MQTT)

Devices authenticate to the MQTT broker using:
- **Client ID**: Device UUID (unique per sensor)
- **Username/Password**: Provisioned during device setup
- **TLS**: Required — CA certificate loaded from device SPIFFS partition

### How to Authenticate (WebSocket)

Pass the JWT access token as a query parameter:

```
ws://localhost:8000/ws?token=eyJhbGciOiJIUzI1NiIs...
```

The server validates the token on connection. Invalid or expired tokens result in immediate disconnection.

## Common Headers

| Header | Required | Description |
|---|---|---|
| `Authorization` | Yes (all except `/auth/*` and `/health`) | `Bearer {access_token}` |
| `Content-Type` | Yes (POST/PUT) | `application/json` |
| `X-Correlation-ID` | Returned | UUID for request tracing — generated by server, returned in response |

---

## Common Errors

All error responses follow this structure:

```json
{
  "error": {
    "code": "ERROR_CODE",
    "message": "Human-readable description",
    "details": {}
  }
}
```

| Status | Code | Meaning | Example |
|---|---|---|---|
| 400 | `VALIDATION_ERROR` | Request body fails validation | `"message": "sampling_rate_hz must be one of: 100, 500, 1000"` |
| 400 | `INVALID_STATE_TRANSITION` | Business rule prevents action | `"message": "Alert must be acknowledged before resolving"` |
| 400 | `HAS_ACTIVE_SENSORS` | Cannot delete machine with sensors | `"message": "Machine has 3 active sensors. Remove them first."` |
| 401 | `UNAUTHORIZED` | Missing or invalid token | `"message": "Invalid or expired access token"` |
| 401 | `INVALID_CREDENTIALS` | Wrong email/password | `"message": "Invalid email or password"` |
| 403 | `FORBIDDEN` | Insufficient role | `"message": "Admin role required"` |
| 404 | `NOT_FOUND` | Resource doesn't exist | `"message": "Machine not found"` |
| 422 | `CANNOT_SELF_MODIFY` | Admin trying to modify self | `"message": "Cannot change your own role"` |

**Query Parameters:**

| Parameter | Type | Required | Default | Description |
|---|---|---|---|---|
| `start` | ISO 8601 | Yes | — | Start of time range |
| `end` | ISO 8601 | Yes | — | End of time range |
| `interval` | string | No | auto | Aggregation interval: `1s`, `1m`, `1h`, `1d` |
| `sensor_id` | UUID | No | all | Filter by specific sensor |

**Auto-interval selection** (when `interval` is omitted):

| Range Duration | Interval Used | Source |

## WebSocket Interface

### Connection

```
ws://localhost:8000/ws?token=eyJhbGciOiJIUzI1NiIs...
```

The server validates the JWT token on connection. Invalid tokens cause an immediate close with code `4001`.

### Client → Server Messages

#### Subscribe to Events

Send after connecting to specify which events you want to receive:

```json
{
  "subscribe": ["fleet", "alerts", "machine:a1b2c3d4-e5f6-7890-abcd-ef1234567890"]
}
```

**Available subscription channels:**

| Channel | Format | Events Received |
|---|---|---|
| `fleet` | `fleet` | Machine status changes for all machines |
| `alerts` | `alerts` | All new and updated alerts |
| `machine:{id}` | `machine:UUID` | Vibration data for a specific machine |

**Default subscriptions** (if no subscribe message sent): `fleet` + `alerts`

### Server → Client Events

#### Vibration Event

Sent when new vibration data is ingested. Rate-limited to 1 per second per machine per client.

```json
{
  "type": "vibration",
  "machine_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "data": {
    "rms_accel_g": 1.45,
    "peak_accel_g": 3.21,
    "rms_velocity_mm_s": 2.3,
    "dominant_freq_hz": 120.5,
    "timestamp": "2026-01-15T14:30:01.000Z"
  }
}
```

**Requires subscription:** `machine:{id}`

#### Alert Event

Sent when a new alert is created or an existing alert's status changes.

```json
{
  "type": "alert",
  "alert": {
    "id": "alert-uuid-001",
    "machine_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "machine_name": "CNC Lathe #4",
    "severity": "critical",
    "alert_type": "threshold_rms",
    "status": "open",
    "value": 7.82,
    "threshold": 5.0,
    "detected_at": "2026-01-15T14:28:00.000Z"
  }
}
```

**Requires subscription:** `alerts`

#### Machine Status Event

Sent when a machine's status changes (e.g., healthy → warning).

```json
{
  "type": "machine_status",
  "machine_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "status": "critical"
}
```

**Requires subscription:** `fleet`

#### Device Health Event

Sent when device health telemetry is received.

```json
{
  "type": "device_health",
  "sensor_id": "s1234567-89ab-cdef-0123-456789abcdef",
  "data": {
    "uptime_s": 86400,
    "free_heap_bytes": 245760,
    "wifi_rssi_dbm": -62,
    "battery_voltage_mv": 3850,
    "firmware_version": "1.2.0",
    "timestamp": "2026-01-15T14:30:00.000Z"
  }
}
```

**Requires subscription:** `fleet`

### Heartbeat

The server sends a WebSocket `ping` frame every 30 seconds. Clients must respond with `pong`. If no pong is received within 60 seconds, the server closes the connection.

### Connection Limits

- Maximum 100 concurrent WebSocket connections
- Vibration events rate-limited to 1/second/machine/client

### WebSocket Close Codes

| Code | Meaning |
|---|---|
| 4001 | Authentication failed (invalid or expired token) |
| 4002 | Connection limit exceeded |
| 1000 | Normal closure |
| 1001 | Server shutting down |

## MQTT Message Interface

### Topic Hierarchy

```
machines/
  {machine_id}/
    vibration     ← Device publishes vibration summaries
    health        ← Device publishes health telemetry
    alerts        ← Device publishes edge-detected alerts
devices/
  {sensor_id}/
    config        ← Backend publishes configuration (retained)
    ota           ← Backend publishes OTA trigger
```

### Topic Details

#### machines/{machine_id}/vibration

**Direction:** Device → Backend
**QoS:** 0 (at-most-once)
**Retain:** No
**Frequency:** ~1 message per second per device
**Max payload size:** 4KB (without FFT), 8KB (with FFT)

**Payload:**
```json
{
  "machine_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "sensor_id": "s1234567-89ab-cdef-0123-456789abcdef",
  "timestamp": "2026-01-15T14:30:01.000Z",
  "rms_accel_g": 1.45,
  "peak_accel_g": 3.21,
  "rms_velocity_mm_s": 2.3,
  "dominant_freq_hz": 120.5,
  "fft_magnitudes": [0.001, 0.003, 0.012, "... (512 floats, optional)"],
  "firmware_version": "1.2.0"
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `machine_id` | UUID string | Yes | Machine this sensor is attached to |
| `sensor_id` | UUID string | Yes | Unique sensor identifier |
| `timestamp` | ISO 8601 UTC | Yes | When the measurement was taken |
| `rms_accel_g` | float | Yes | RMS acceleration in g |
| `peak_accel_g` | float | Yes | Peak acceleration in g |
| `rms_velocity_mm_s` | float | Yes | RMS velocity in mm/s |
| `dominant_freq_hz` | float | Yes | Dominant frequency in Hz |
| `fft_magnitudes` | float[512] | No | FFT magnitude spectrum (if `include_fft` config is true) |
| `firmware_version` | semver string | Yes | Current firmware version |

## Versioning

- **Current version:** v1 (no version prefix in URL — all endpoints under `/api/`)
- **MQTT schema version:** Implicit in firmware version — breaking changes require firmware OTA update
- **WebSocket protocol:** No versioning — changes are additive (new event types, new fields)
- **Breaking change policy:** REST API breaking changes will be deployed under `/api/v2/` with 6-month deprecation of `/api/v1/`
